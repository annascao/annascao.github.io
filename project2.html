Track-Atack

Python Raspberry Pi OpenCV Flask

Track-Atack is a rubber band-shooting gun featuring aim assist / target tracking made in 24 hours for nwHacks. Our goal was to enable users to select a target via a web app and have the system accurately aim and shoot at the chosen target.

The setup featured a static base housing two servos, each controlling a mounted mechanism to adjust the pitch and yaw angles of the rubber band shooter. A camera was used to capture the target's position, with the data processed by a web app to identify precise coordinates. These coordinates were transmitted to a Raspberry Pi, which ran a control system to calculate the required adjustments for each servo.

This was a fun learning experience where we got to try out new technologies and tools and it challenged us to think on our feet with limited time! We hope to continue and refine this project!

Software

Our system relied on a coordinated software architecture split between the Raspberry Pi and a laptop.

  1. On the Laptop:
    • The camera feed was processed to detect objects.
    • Users interacted with the visualized feed to select a target.
    • The selected target’s coordinates were sent to the Raspberry Pi via the Flask API.
  2. On the Raspberry Pi:
    • The Flask server received the coordinates.
    • The control loop adjusted the servos to aim at the target based on the provided coordinates.

The laptop hosted the user interface, where a Python-based application utilized YOLOv5 for object detection and SORT for object tracking. Users could interact with the interface to select a target in the live video feed, streamed from a usb camera. OpenCV was used to render the video feed with real-time visual overlays, including bounding boxes and center points. The selected target's coordinates (x, y) were computed and sent via a REST API using HTTP POST requests to the Raspberry Pi.

These coordinates were processed in a control system that calculated the adjustments for the servos controlling the rubber band shooter’s pitch and yaw.

Mechanical

Things We Learned / Issues We Ran Into

  • Initially, we wanted the object detection and video processing to be done on the Pi. We then wanted to video stream to the web-app. Unfortunately, with the Pi 3, we were struggling to run the entire project and there were computational limitations. Therefore, offloaded the heavy computational tasks to the laptop.
  • The servos moved in bigger intervals than expected because we used software-generated PWM signals. The lack of precise timing meant the signals were not perfect and the servos jiggled back and forth a bit.
  • The rubber bands we brought were too thick. Luckily, there was Bánh mì for dinner and we were able to collect the rubber bands from the sandwich packaging which were a perfect size!
